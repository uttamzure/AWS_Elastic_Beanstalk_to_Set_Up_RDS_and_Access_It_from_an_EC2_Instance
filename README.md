## 📌 Project-1

# AWS_Elastic_Beanstalk_to_Set_Up_RDS_and_Access_It_from_an_EC2_Instance

## 🎯 Objective
Learn how to deploy an application environment using AWS Elastic Beanstalk, provision an integrated Amazon RDS database, and access the RDS instance securely from an external EC2 instance within the same VPC. 
-

##  Technologies Used
- AWS Elastic Beanstalk
- Amazon RDS (MySQL)
- Amazon EC2
- VPC, Security Groups
- SSH & MySQL Client

---

##  Architecture Diagram

![Architecture](./IMG/architecture%20project%201.png)

---
##  Key Workflow (Short & Clean)
- Deploy app using AWS Elastic Beanstalk Amazon Web Services

- Enable RDS auto-creation with Beanstalk
- Keep Beanstalk + RDS in same VPC
- Set Security Groups (only Beanstalk + EC2 access)
- Launch EC2 instance in same VPC
- SSH into EC2 instance
- Install DB client (MySQL / PostgreSQL)
- Connect using RDS endpoint + credentials
- Run DB commands (SHOW DATABASES, SELECT, etc.)

---

## Prerequisites

- AWS EC2 instance with SSH access
- AWS RDS MySQL instance 
- Security Group allowing:
  - Port 22 (SSH)
  - Port 3306 (MySQL)
- RDS endpoint, username, and password

---



### (1) RDS Setup via Elastic Beanstalk

In this project, Amazon RDS was automatically provisioned during the AWS Elastic Beanstalk environment creation process.



- RDS instance created automatically by Elastic Beanstalk
- Deployed in same VPC as application
- Endpoint and credentials generated by AWS

### (2)💻 EC2 Access to RDS

A separate EC2 instance was used to connect to the RDS database.

- EC2 launched in same VPC
- SSH connection established using key pair
- MySQL client installed on EC2
- Connected to RDS using endpoint and credentials
- Security group restricted access to EC2 only
